How it works

scriptfix is a post-processor, not an OCR engine. Tesseract turns the image into characters; scriptfix applies the linguistic rules Tesseract can't.

1

Image → Tesseract

Run Tesseract with JSON output. It returns characters, confidence and bounding boxes.

→
2

Confidence routing

≥85% passes through untouched, <60% is flagged for review, the middle band is corrected.

→
3

Evidence-gated repair

A fix is applied only if it lowers script-validity badness — so correct text is never changed.

→
4

Clean Unicode

Corrected text, a per-fix report and preserved layout metadata.

Sihari reordering

The dependent vowel ਿ is written before its consonant but must be encoded after it. scriptfix moves it back.

Nukta canonicalisation

A nukta emitted after a vowel sign (ਸਾ਼) is reordered to the canonical consonant+nukta+vowel (ਸ਼ਾ).

Never corrupts good text

Corrections require validity evidence. Already-correct Unicode rounds-trips byte-for-byte — enforced by CI.

Validity report

Orphaned matras, impossible sequences and out-of-script codepoints are surfaced with severity.

Batch + learning

Parallel batch processing and a SQLite store that promotes repeatedly-confirmed corrections.

Layout preserved

Bounding boxes flow through end-to-end so downstream tools can rebuild the page.

Six scripts, one pipeline

Shared rule engine with per-script YAML rules and inheritance via extends.

Get started

Pure-Python, MIT-licensed. Tesseract is a peer dependency, not a runtime requirement.

Install
pip install scriptfix
Run Tesseract → scriptfix
tesseract page.tif out --oem 1 --psm 6 json
scriptfix correct --input out.json \
  --lang gurmukhi --output ./results
Batch a folder
scriptfix batch --input-dir ./pages \
  --lang devanagari --workers 4
